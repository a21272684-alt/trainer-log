## 7. AI 수업일지 월 사용 한도 시스템

**위치:** 트레이너 포털 > 회원 상세 > 수업일지 작성  
**기획 배경:** 트레이너별 무료 플랜 월 20회 제한, 초과 시 자동 차단

### DB 마이그레이션 (`028_ai_usage_limit.sql`)

```sql
-- trainers 에 플랜 컬럼 추가
ALTER TABLE trainers
  ADD COLUMN IF NOT EXISTS plan_type       text    NOT NULL DEFAULT 'free'
    CHECK (plan_type IN ('free','pro','unlimited')),
  ADD COLUMN IF NOT EXISTS ai_monthly_limit integer DEFAULT 20;

-- 월별 사용량 추적 테이블
CREATE TABLE IF NOT EXISTS ai_usage (
  id         uuid    DEFAULT gen_random_uuid() PRIMARY KEY,
  trainer_id uuid    NOT NULL REFERENCES trainers(id) ON DELETE CASCADE,
  used_year  integer NOT NULL,
  used_month integer NOT NULL,
  count      integer NOT NULL DEFAULT 0,
  updated_at timestamp DEFAULT now(),
  UNIQUE (trainer_id, used_year, used_month)
);

-- RPC 3종
-- get_ai_usage(trainer_id)     → 사용량 조회 (blocked 여부 포함)
-- consume_ai_credit(trainer_id) → 성공 후 카운트 +1
-- admin_set_trainer_plan(...)   → 어드민 플랜 변경
```

> **버그 수정:** `SELECT COALESCE(count,0) INTO v_count` 가 행이 없을 때 NULL 반환하는 문제  
> → 서브쿼리 COALESCE 방식으로 수정:  
> `v_count := COALESCE((SELECT count FROM ai_usage WHERE ...), 0);`

### 프론트엔드 변경

**추가 state:**
```js
const [aiUsage, setAiUsage] = useState(null)   // { plan, limit, used, remaining, blocked }
const [showLimitModal, setShowLimitModal] = useState(false)
```

**흐름:**
1. 로그인 → `loadAiUsage()` → `get_ai_usage` RPC 호출 → `aiUsage` 세팅
2. AI 생성 버튼 클릭 → `generateLog()` 내부에서 한도 재확인
3. `blocked: true` → 차단 모달 표시, 생성 중단
4. 생성 성공 → `consume_ai_credit` RPC 호출 → `aiUsage` 업데이트

**UI:**
- 파스텔 컬러 프로그레스바 (8px, 발광 효과)
  - 여유 (8회↑): 파스텔 그린 `#86efac`
  - 주의 (4~7회): 파스텔 옐로 `#fcd34d`
  - 위험 (1~3회): 파스텔 핑크 `#f9a8d4`
- 텍스트: `✦ 이번 달 N회 남았어요` / `X / 20회`
- 한도 초과 모달: 다음 달 초기화 안내 + 업그레이드 이메일 링크

**관련 파일:** `src/pages/TrainerApp.jsx`, `supabase/migrations/028_ai_usage_limit.sql`
