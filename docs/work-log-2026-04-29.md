## 1. 구글 OAuth 트레이너 등록 오류 수정

### 배경
구글 소셜 로그인 후 "트레이너 등록 완료" 버튼 클릭 시 오류 발생.

### 원인
`trainers.phone` 컬럼이 `NOT NULL` 제약이 걸려 있는데,
OAuth 등록 시 phone 값 없이 INSERT → Postgres 제약 오류.

### 수정 내용

**DB 마이그레이션 (`038_phone_nullable.sql`)**
```sql
ALTER TABLE trainers ALTER COLUMN phone DROP NOT NULL;
ALTER TABLE trainers ALTER COLUMN phone SET DEFAULT '';
```

**`register()` 함수 (TrainerApp.jsx)**
- `insert()`에 `phone: ''` 명시적 추가
- `037_oauth_auth.sql` 미실행(auth_id 컬럼 없음) 시 phone만으로 폴백 등록 처리
- `_loginWithRecord` 내부 예외가 등록 흐름을 막지 않도록 `try/catch` 분리
- 등록 화면 하단에 실제 에러 메시지를 빨간 박스로 표시 (`regError` state)
- `console.error` 단계별 로깅 추가

```js
// 폴백 처리: auth_id 컬럼 없을 때
if (insertErr.message?.includes('auth_id') || insertErr.code === '42703') {
  const { data: ins2 } = await supabase
    .from('trainers').insert({ name: regName, phone: '' }).select().single()
  await _loginWithRecord(ins2)
  return
}
```
