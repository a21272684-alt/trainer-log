### 2. 회원포털 수업일지 미표시 오류 수정 (2건)

#### 2-1. `sendKakao()` silent fail 문제

**원인**
Supabase JS 클라이언트는 insert 실패 시 예외를 throw하지 않고 `{ data, error }` 형태로 반환하는데, 기존 코드가 `error`를 전혀 확인하지 않아 insert 실패가 눈에 띄지 않았음.

실제 원인: `004_report_exercises.sql` 마이그레이션 미적용으로 `report_id`, `exercises_data` 컬럼이 DB에 없어 매번 insert 실패.

**수정 내용**
```js
// 수정 전
await supabase.from('logs').insert({ ... })

// 수정 후
const { error: logErr } = await supabase.from('logs').insert({ ... })
if (logErr) throw new Error('일지 저장 실패: ' + logErr.message)
```

**DB 조치** (Supabase SQL Editor 실행 필요)
```sql
ALTER TABLE logs ADD COLUMN IF NOT EXISTS report_id text;
ALTER TABLE logs ADD COLUMN IF NOT EXISTS exercises_data jsonb;
```

#### 2-2. 회원포털 수업일지 탭 미갱신 문제

**원인**
`loadAll()`이 로그인 시 1회만 실행되어, 이후 트레이너가 생성한 일지가 회원 화면에 반영되지 않음.

**수정 내용**
```js
// 수업일지 탭 진입 시마다 최신 목록 재로드
useEffect(() => {
  if (tab === 'logs' && member) {
    supabase.from('logs').select('*')
      .eq('member_id', member.id)
      .order('created_at', { ascending: false })
      .then(({ data }) => setMemberLogs(data || []))
  }
}, [tab])
```
