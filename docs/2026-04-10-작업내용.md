## 3. 회원 정지(기간 홀딩) 기능 추가

### 정지 버튼
- 각 회원 상세 탭 우측 상단, 결제 버튼 왼쪽에 배치
- 정지 중일 때 주황색 `⏸ 정지중` 표시

### 정지 모달 구성
- **기간 설정** — 시작일 / 종료일, 총 정지 일수 자동 계산 표시
- **상품 선택** — 등록된 상품 중 선택 (선택사항)
- **정지 사유** — 텍스트 메모
- **사진 첨부** — 이미지 파일 선택 + 미리보기 (✕ 제거 가능)
- **정지 이력** — 기존 홀딩 목록 + 개별 해제 버튼

### 자동 상태 연동
- 정지 등록 → `suspended = true` 처리
- 이력 전체 해제 → `suspended = false` 자동 복원

### DB 마이그레이션
- `009_member_holds.sql` — `member_holds` 테이블 생성
- `010_hold_photos_bucket.sql` — Supabase Storage `hold-photos` 버킷 생성

- 회원 상세의 수업일지 / 출석부 / 건강기록 옆에 **`⏸ 정지기록`** 탭 추가

## 5. 정지기록 버그 수정

### 원인
- `addHold`에서 `supabase.insert()` 결과의 `error`를 체크하지 않아 INSERT 실패해도 `suspended=true`만 설정됨
- RLS 정책이 `auth.uid()`를 기반으로 설정되어 있으나 앱이 Supabase Auth 미사용 → INSERT/SELECT 전체 차단

### 수정 내용
- `addHold` — `insertErr` 체크 후 throw, 실패 시 `suspended=true` 설정 안 함
- `loadHolds` — error 시 기존 holds 상태 유지
- `deleteHold` — DELETE error 체크 추가
- `useEffect` 추가 — 정지기록 탭 진입 시 자동 reload

### DB 마이그레이션
- `011_fix_holds_rls.sql` — `member_holds` RLS 정책을 앱 인증 방식에 맞게 수정 (auth.uid() → 전체 허용, 앱 레벨 필터링)
