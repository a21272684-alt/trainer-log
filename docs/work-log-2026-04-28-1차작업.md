관리자 포털의 랜딩페이지 탭 구조를 2단계 계층으로 재편하고,

## 1. 관리자 포털 — 랜딩페이지 탭 2단계 구조 개편

### 변경 전
- 랜딩페이지 탭 하위에 11개 서브탭이 단일 레벨로 나열

### 변경 후
- **레벨 1 (포털 선택)**: `🌐 메인 랜딩` / `🤝 커뮤니티 랜딩` / `🏢 CRM 랜딩`
- **레벨 2 (섹션 선택)**: 선택된 포털에 맞는 서브탭 표시

### 기술 구현
- `LANDING_PORTALS` / `LANDING_TABS` 상수 추가
- `landingSite` state (`'main' | 'community' | 'crm'`)
- `switchLandingSite(site)` 헬퍼 함수 (landingSite + subTab 동시 초기화)
- 기존 11개 메인 랜딩 섹션에 `landingSite === 'main'` 가드 추가

**파일**: `src/pages/AdminPortal.jsx`

## 6. 포털 버튼 ON/OFF 기능

### 목적
랜딩페이지 GET STARTED 섹션의 각 포털 카드 노출 여부를 관리자가 제어

### 관리자 포털
- `메인 랜딩 > 포털 버튼` 탭 신규 추가
- 4개 포털 (트레이너 앱 / 회원 포털 / 커뮤니티 / 헬스장 CRM) 개별 토글
- 토글 클릭 즉시 저장 (별도 저장 버튼 없음)
- ON/OFF 상태를 색상 뱃지 + 카드 스타일로 시각화

### 랜딩페이지 반영
| 상태 | 표시 |
|------|------|
| ON | 기존과 동일한 클릭 가능 Link 카드 |
| OFF | 회색(grayscale) 처리 + `cursor: not-allowed` + **🔜 준비중** 뱃지 |

### Supabase key
```json
// key: 'landing_portal_buttons'
{
  "trainer": true,
  "member": true,
  "community": true,
  "crm": false  // 예시: CRM OFF
}
```

**파일**: `src/pages/AdminPortal.jsx`, `src/pages/Landing.jsx`
