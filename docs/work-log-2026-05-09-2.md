### PR #28 — admin 기능 게이트 + 랜딩 silent fail rollback (`fix(admin)`)

**배경:**
사용자가 "admin > 기능 게이트 ON/OFF 토글이 안 먹는다" 보고 → audit 결과 **사용자 기억이 정확** 함. 토글 핸들러는 정상 연결됐으나 optimistic update 후 catch 에서 rollback 안 함 → save 실패 시 UI 는 토글된 상태 + 에러 토스트만 뜨고 새로고침 시 DB 의 이전 값으로 회귀.

**처리:**
1. **`saveFeatureGates`** — prev 캡처 + 실패 시 rollback + 명확한 토스트
2. **`saveLandingField` 헬퍼 신규** — 모든 랜딩 save 함수가 동일 패턴
3. **18개 `saveLanding*` 함수 리팩터** — 헬퍼 사용 (메인 7 + CRM 4 + 커뮤니티 1 + 통계/후기 등 6)
4. **`saveMemberLimit` edge case 주석** — 입력 시점마다 setMemberLimit 가 state 갱신하므로 저장 실패 시 입력값 그대로. 새로고침 시 복원됨. 베타 OK.

**효과:** 기능 게이트 ON/OFF + 19개 랜딩 섹션 편집 모두 실패 시 UI 자동 복원 + 사용자 알림. 기존엔 토스트만 뜨고 새로고침 시 원복 → "안 먹힌다" 인식.

### PR #29 — admin 세션 영속화 (`fix(admin)`)

**배경:**
사용자 보고 — admin 새로고침 시 매번 로그인 화면 회귀. 트레이너/회원 앱은 Supabase Auth JWT 자동 복원이 있으나, admin 은 ID/PW 클라이언트 검증이라 별도 영속 메커니즘 부재했음.

**처리:**
- `sessionStorage` 에 `{ id, ts }` 저장 (탭 종료 시 자동 만료)
- 8시간 TTL — stale 세션 자동 무효화
- `VITE_ADMIN_ID` 환경변수 변경 시 자동 로그아웃 (id 불일치)
- login 성공 시 sessionStorage 쓰기, logout 시 지우기
- useState 초기값을 lazy init 으로 마운트 즉시 로그인 상태 복원

**보안 트레이드오프:**
- localStorage 가 아닌 sessionStorage — 탭 닫으면 자동 만료
- 8시간 TTL 추가 보호
- 본질적 보안 수준은 기존 ID/PW 클라이언트 검증과 동일 (Phase E 의 Edge Function + httpOnly cookie 마이그레이션 시 강화)

### PR #30 — CRM 권한 키 admin↔CRM 동기화 (`fix(admin/crm)`)

**배경:**
사용자가 "admin 의 CRM 활성화 / 리드관리 / 고객노트 / 팔로업 / 데이터 내보내기 토글이 정상 작동 안 한다" 의심 → audit 결과 **사용자 의심이 정확함**. admin 의 5개 토글 키 모두 CRM 코드에서 한 번도 읽히지 않는 silent fail.

**근본 원인:**
- admin 측 토글: `enabled`, `lead_management`, `client_notes`, `follow_up`, `data_export` (5개)
- CRM 측 실제 키 (PERM_DEFS): `crm_access`, `view_all_members`, `manage_products`, `view_settlement` (4개)
- 두 집합이 **완전히 다름** → admin DB 저장은 되지만 CRM 어디에서도 안 읽음

**처리:**
1. **CRM_FEATURES 4개 가짜 키 제거 → 실제 3개 키로 교체** (`view_all_members` / `manage_products` / `view_settlement`)
2. **`updateCrmEnabled` 재배선** — `enabled` 가 아닌 `crm_access` 토글 (legacy `enabled` 도 동시 set 으로 backward compat)
3. **CRM `GymPortal.jsx` 에 `crm_access` 게이트 추가**:
   - `role === 'owner'` → 무조건 통과 (자기 헬스장 보호)
   - 그 외 직원 → `crm_access || enabled` true 여야 진입
   - 차단 시 🔒 access_denied 화면 (다른 계정 로그인 버튼)

**효과:** admin 의 모든 토글이 실제 CRM 동작에 영향. owner 는 항상 진입 가능. 직원은 admin 차단 시 명확한 안내 화면.
