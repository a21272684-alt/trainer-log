## 1. DB 마이그레이션 — `diet_logs` 테이블

기존 `health_records.diet_note` (자유 텍스트)를 대체하는 구조화된 테이블.

```sql
CREATE TABLE diet_logs (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id       UUID NOT NULL REFERENCES members(id) ON DELETE CASCADE,
  record_date     DATE NOT NULL,
  meal_type       TEXT NOT NULL CHECK (meal_type IN ('breakfast','lunch','dinner','snack')),
  food_name       TEXT NOT NULL,
  amount_g        DECIMAL(10,2) NOT NULL DEFAULT 100,

  -- 영양소: g당 정규화 값으로 저장
  calories_per_g  DECIMAL(10,6),
  protein_per_g   DECIMAL(10,6),
  carbs_per_g     DECIMAL(10,6),
  fat_per_g       DECIMAL(10,6),
  fiber_per_g     DECIMAL(10,6),
  sodium_per_g    DECIMAL(10,6),  -- mg/g 단위
  sugar_per_g     DECIMAL(10,6),

  photo_url       TEXT,
  ai_recognized   BOOLEAN NOT NULL DEFAULT false,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**정규화 설계 원칙:** 1인분·100g당·1회 제공량 등 데이터마다 단위가 다르므로, AI 응답을 받는 즉시 내부적으로 **g당 값으로 환산**해 저장. 실제 섭취량 계산 시 `amount_g × per_g` 로 일관성 보장.

**Storage 버킷:** `diet-photos` (public) — Supabase 대시보드 / API로 생성.

## 2. AI 템플릿 — `src/lib/ai_templates.js`

### 추가 함수

#### `buildFoodVisionParts(base64Data, mimeType)`
Gemini Vision 멀티파트 요청 parts 배열 반환. 이미지 + 프롬프트로 구성.
- AI에게 **100g당 기준**으로 응답하도록 지시
- 응답 형식: JSON (food_name, estimated_amount_g, per_100g 객체, confidence)

#### `parseFoodVisionResult(text)`
Gemini 응답 텍스트에서 JSON 파싱 후 **g당으로 정규화** (÷100).
- 나트륨: mg/100g → mg/g (mg 단위 유지)
- 나머지 영양소: g/100g → g/g

## 3. 회원포털 식단기록 탭 — `MemberPortal.jsx`

### 신규 state
- `trainerApiKey` — 로그인 시 `trainers.api_key` 자동 로드 (Gemini 호출용)
- `dietLogs` — 선택 날짜의 diet_logs 목록
- `dietDate` — 조회 날짜
- `showFoodModal` / `foodMealType` / `foodName` / `foodAmountG` — 모달 상태
- `foodCalPerG` ~ `foodSugarPerG` — 영양소 입력값 (g당 내부 값)
- `foodPhotoFile` / `foodPhotoPreview` / `foodAiLoading` / `foodAiConfidence` — 사진 AI 상태

### 주요 함수

| 함수 | 역할 |
|------|------|
| `loadDietLogs(date)` | 해당 날짜 식단 로그 fetch |
| `openFoodModal(mealType)` | 모달 열기 + 상태 초기화 |
| `recognizeFoodFromPhoto(file)` | 사진 → FileReader → Gemini Vision → 폼 자동 입력 |
| `addFoodItem()` | 사진 업로드(선택) + diet_logs insert |
| `removeFoodItem(id)` | 항목 삭제 |
