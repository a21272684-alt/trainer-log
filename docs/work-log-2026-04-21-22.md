### 2. 프로필 사진 업로드 오류 수정

**원인**
`033_trainer_profile.sql` 마이그레이션이 Supabase에 미적용 상태로, 버킷·컬럼·정책 모두 부재하여 업로드 즉시 실패.

기존 마이그레이션 SQL의 `DO $$ ... IF NOT EXISTS ... EXECUTE` 패턴이 `pg_policies` schemaname 미필터로 정책 중복 오류 가능성도 있었음.

**수정 내용**
- `033_trainer_profile.sql` — 검증된 `diet-photos` 동일 패턴으로 재작성
  ```sql
  CREATE POLICY "allow_all_trainer_photos" ON storage.objects
    FOR ALL
    USING (bucket_id = 'trainer-photos')
    WITH CHECK (bucket_id = 'trainer-photos');
  ```
- `uploadProfilePhoto()` 함수 오류 처리 강화
  - `Bucket not found` → "SQL 실행 안내" 안내 메시지
  - `column profile_photo_url` → "DB 컬럼 없음" 안내 메시지
  - 이미지 파일 형식 사전 검증 (JPG·PNG·WebP·GIF)
  - `contentType` 명시 업로드
