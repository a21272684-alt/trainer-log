### Bug #1 (오인) — 회원 미등록자도 회원앱 진입 가능

**진단 결과: false alarm.** 코드 audit 결과 `member_resolve_self` RPC 가 NULL 반환 시 클라이언트가 자동 로그아웃 + "등록된 회원 정보가 없어요" 토스트로 정상 차단. dev 데이터에 본인 row 가 있었던 케이스 또는 stale data 영향으로 추정. **수정 불필요.**

### Bug #2 (실재) — 트레이너 가입 시 admin 승인 없이 자동 등록

**진단 결과: 실제 문제 — 옛 의도 (자유 가입 + 후일 정리) 가 베타 단계에는 부적합.**

`trainer_resolve_or_create` RPC 의 분기 (c) 가 신규 OAuth 사용자에게 trainers row 를 자동 INSERT. `admin_register_trainer` RPC (052) 는 있지만 OAuth signup 흐름에서 호출 안 됨.

`approval_status` 컬럼 (040) 은 존재하지만 gym 가입 승인 (CRM) 용이고 트레이너 가입에는 무관.

## 발생한 함정 (1개)

### 빈 trainer row 함정 — Postgres composite type + RETURN NULL

**증상:**
검증 3 (신규 OAuth) 흐름에서 사용자가 OAuth 직후 트레이너 메인 화면으로 빈 상태 진입. 콘솔에 `trainer_id=eq.null` 로 모든 후속 쿼리 400 에러.

**원인:**
Postgres 함수 `RETURNS trainers` (composite type) 가 `RETURN NULL` 할 때 PostgREST/supabase-js 가 그것을 **`{id: null, name: null, email: null, ...}` 형태의 빈 객체**로 변환. JS `if (row)` 가 truthy 평가되어 `_loginWithRecord(row)` 로 빠짐.

**Fix:**
```js
// 변경 전:
if (row) { await _loginWithRecord(row); return }
// 변경 후:
if (row && row.id) { await _loginWithRecord(row); return }
```
3군데 (handleAuthUser 진입 + handleAuthUser already_trainer retry + register already_trainer retry).

검증 1 (기존 트레이너) 가 통과한 이유: 그 경우 `row.id` 가 valid uuid 라 정상.
