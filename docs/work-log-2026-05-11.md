## 발생한 함정 (2개 — 동일 패턴)

### 함정 #1 — 빈 trainer row (`trainer_resolve_or_create`)

**증상:**
검증 3 (신규 OAuth) 흐름에서 사용자가 OAuth 직후 트레이너 메인 화면으로 빈 상태 진입. 콘솔에 `trainer_id=eq.null` 로 모든 후속 쿼리 400 에러.

**원인:**
Postgres 함수 `RETURNS trainers` (composite type) 가 `RETURN NULL` 할 때 PostgREST/supabase-js 가 그것을 **`{id: null, name: null, email: null, ...}` 형태의 빈 객체**로 변환. JS `if (row)` 가 truthy 평가되어 `_loginWithRecord(row)` 로 빠짐.

**Fix:**
```js
// 변경 전:
if (row) { await _loginWithRecord(row); return }
// 변경 후:
if (row && row.id) { await _loginWithRecord(row); return }
```
TrainerApp.jsx 3군데 (handleAuthUser 진입 + handleAuthUser already_trainer retry + register already_trainer retry).

### 함정 #2 — 빈 member row (`member_resolve_self`) — **보안 영향**

**증상:**
박성인 (sbag8798) 사용자가 트레이너 앱 가입은 미완료 상태인데 회원앱(ownapp.kr/member) 에 진입 가능. 트레이너가 그를 회원으로 추가하지 않았음에도.

**원인:**
함정 #1 과 정확히 동일 패턴 — `member_resolve_self` 도 `RETURNS members` (composite type) + RETURN NULL → 빈 객체 변환. MemberPortal.jsx 의 `if (row) { _loginWithRecord(row) }` 가 빈 객체에서도 truthy 평가 → 회원앱 진입 허용.

**Fix:**
MemberPortal.jsx 의 `handleAuthUser` 에 동일한 `if (row && row.id)` 가드 추가. 매칭 실패 시 다음 분기로 빠져 "등록된 회원 정보가 없어요" 토스트 + 자동 로그아웃 (의도된 흐름).

**영향:**
- 정상 등록 회원 (members 에 row 있음): row.id valid → 영향 0
- 트레이너 미등록 사용자 (박성인 등): 회원앱 진입 차단 (의도된 보안 동작)

**중요:** 이 두 함정은 **세션 초반 코드 audit 만으로는 발견 불가** — RPC 가 의도적으로 NULL 반환하는 것 자체는 정상이지만, PostgREST 의 composite 타입 변환 규칙이 그 NULL 을 빈 객체로 보내는 것이 함정. **dev/운영에서 실제 신규 OAuth 사용자 케이스 검증해야 발견**됨.

-- 가입 흐름 어딘가에 갇힌 사용자 점검
SELECT u.id, u.email, u.created_at, u.last_sign_in_at,
       u.raw_user_meta_data->>'full_name' as name
FROM auth.users u
WHERE u.email IS NOT NULL
  AND NOT EXISTS (SELECT 1 FROM trainers t WHERE t.email = u.email)
  AND NOT EXISTS (SELECT 1 FROM trainer_signup_requests r WHERE r.email = u.email)
  AND NOT EXISTS (SELECT 1 FROM members m WHERE m.email = u.email)
ORDER BY u.last_sign_in_at DESC NULLS LAST
LIMIT 50;
