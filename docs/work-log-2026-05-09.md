### PR #23 — Phase D-1 Sentry 에러 트래킹 (`feat(observability)`)

**배경:**
3개 앱 (portal/admin/crm) 의 사용자 브라우저 측 JS 에러를 자동 수집·통지. 베타 단계엔 사용자 신고 외엔 버그 인지가 어려움.

**구성:**
- `packages/shared/lib/sentry.js` (신규) — `initSentry(appName)` 헬퍼
  - DSN 미설정 시 no-op (앱 동작 영향 0)
  - `sendDefaultPii=false` (한국 PIPA 친화 — IP/UA 자동 수집 차단)
  - `tracesSampleRate=0` / replays 둘 다 0 (5K 무료 이벤트 한도 보호)
  - `beforeSend` 에서 `user.email/username/ip_address` 및 `ui.input` breadcrumb 정리
  - `ignoreErrors`: ResizeObserver/NetworkError 등 알려진 노이즈 차단
  - `tag.app` 으로 portal/admin/crm 구분 (단일 Sentry 프로젝트에서 분리 분석)
- `apps/{portal,admin,crm}/src/main.jsx` — `initSentry('<app>')` 호출
- `packages/shared/package.json` — `@sentry/react` peerDep 추가
- 각 앱 `package.json` — `@sentry/react ^9` 의존성 추가
- `.env.example` — `VITE_SENTRY_DSN` 주석 보강
- `apps/portal/src/pages/Privacy.jsx` — 처리위탁 표에 Functional Software, Inc. (Sentry) 행 추가

**가입 + 설정 (사용자 작업):**
1. https://sentry.io GitHub 계정 가입 → Developer 무료 플랜 (5K 이벤트/월)
2. Data Storage Location: United States 선택 (한국에서 latency 짧음)
3. Platform: React, 프로젝트명 `javascript-react`
4. DSN 발급 → main repo `.env` 의 `VITE_SENTRY_DSN` 에 주입

**종단 검증 완료:**
- 브라우저 콘솔에 `setTimeout(() => { throw new Error("sentry test") }, 0)` 입력 → 1~2분 내 Sentry Issues 페이지에 "Error: sentry test" 도착 확인
- Sentry 가 init 됐어도 직접 콘솔 throw 는 dev tools eval 래퍼가 잡아 window.onerror 까지 안 가는 함정 발견 → setTimeout 으로 비동기 throw 시 정상 캡처되는 것으로 우회

**후속 (선택):**
- `Sentry.ErrorBoundary` 로 React 렌더 에러 catch + 사용자 친화 fallback UI
- 알림 설정 (Sentry → Settings → Alerts) — 신규 이슈 발생 시 이메일·슬랙
