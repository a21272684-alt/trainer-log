**변경:**

- 수정: `apps/portal/src/pages/Landing.jsx` — 4개 포털 카드 + 푸터 링크 모두 IIFE 로 3-way 정규화. 'hidden' 이면 SlideCard 자체 미렌더

베타 권장: 트레이너/회원 = 표시, 커뮤니티/CRM = 숨김.

### PR #33 — 포털 버튼 3-way normalize fix (`fix`)

**버그:**
PR #32 머지 후 사용자가 admin 에서 어떤 버튼을 클릭해도 랜딩에 항상 "준비중" 표시.

**원인:**
Landing.jsx 의 `normalizePortals` 헬퍼가 옛날 boolean 강제 변환 (`toStrictBool`) 그대로 → admin 이 string `'show'/'coming_soon'/'hidden'` 저장해도 모두 `false` (false == 'coming_soon') 로 변환.

**Fix:**
3-way pass-through 로 변경:
- VALID_STATES = `['show', 'coming_soon', 'hidden']`
- string 유효 시 그대로 보존
- boolean (legacy) 도 그대로
- 알 수 없는 값 → `'show'` 기본 (안전한 기본 노출)

### PR #34 — TrainerApp N+1 perf fix (`perf`)

**증상:**
사용자가 "트레이너 앱이 렉" 보고. perf audit 결과 `members.find()` 가 5군데 (`reduce`/`map` 안에) 호출되어 회원 100명 + 로그 500건 시나리오에서 **매 렌더 50,000회 비교**.

**Fix:**
TrainerApp 컴포넌트 최상단에 `useMemo` 로 `memberById = new Map(members.map(m => [m.id, m]))` 생성. 5군데에서 `.find()` → `.get()` 교체:
- renderRevenue 의 weekRevenue / monthRevenue (3670, 3671)
- displayList 일지 리스트 (4773)
- 미확인 일지 리스트 (6952)
- 확인 완료 일지 리스트 (6972)

**복잡도:** O(N×M) → O(N) (~80배 빠름).

### PR #35 — Vercel SPA fallback (`fix`)

**증상:**
ownapp.kr/trainer 직접 접근 또는 OAuth 콜백 도착 시 Vercel "404 NOT_FOUND" 반환.

**원인:**
React Router (BrowserRouter) 는 클라이언트 측 라우팅. Vercel 이 정적 파일 `/trainer` 찾지 못해 404. 모든 경로를 `/index.html` 로 rewrite 해야 React Router 가 처리.

**Fix:**
신규 `apps/portal/vercel.json`:
```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

머지 후 자동 재배포 → 직접 라우트 접근 + OAuth 콜백 모두 정상.

## 의사결정 — admin Vercel 배포 보류

### 배경

own-admin Vercel 프로젝트 생성 후 Deployment Protection 설정 단계에서 발견:
- Vercel **Hobby 플랜**: Vercel Authentication 이 Preview 만 보호, Production 은 항상 공개
- admin Production URL 노출 시 누구나 admin 로그인 화면 도달 가능
- admin 의 ID/PW + ADMIN_DB_TOKEN 이 클라이언트 빌드에 박힘 (devtools 추출 가능)

### 3가지 옵션

| 옵션 | 비용 | 보안 | 편의 |
|---|---|---|---|
| A: Hobby + 강력 비번 + 랜덤 subdomain | 무료 | 중 | 중 |
| B: Vercel Pro 업그레이드 | $20/월 | 강 | 강 |
| C: Vercel 미배포 + 로컬 전용 | 무료 | 최강 (인터넷 노출 0) | 약간 불편 |

### 결정 — 옵션 C

베타 단계 본인만 admin 사용 + 사용자 30명 이하 환경에서 외부 attack surface 자체를 만들지 않는 게 가장 안전.

**일상 흐름:**
```
admin 작업 필요 시
  → cd C:\...\trainer-log
  → npm run dev:admin
  → localhost:3010 자동 오픈
  → 작업 완료 후 Ctrl+C
```

**향후 재검토:**
정식 출시 (사용자 100명+ 또는 외부 운영자 추가) 시점에 (a) Vercel Pro 업그레이드 또는 (b) Edge Function + httpOnly cookie 본격 보안 도입.

own-admin Vercel 프로젝트는 생성 직후 삭제 (Settings → General → Delete Project).
